# v1.2 Plan (Draft)

状态：`PLANNING`
更新时间：2026-04-13

## 目标
1. 提升生产可观测性，降低“降级成功但质量不可见”的风险。
2. 提升真实环境一致性（依赖、门禁、回归矩阵）。
3. 降低流程复杂度，收敛配置与失败处理策略。

## 范围（候选）

### A. 运行时可靠性
- 引入“严格模式”开关：禁止任何非显式降级路径。
- 为关键阶段增加结构化错误码（writer/reviewer/polisher/data_analyst）。
- 增加阶段级超时与重试策略（可配置上限）。

### B. 依赖与环境基线
- 明确 `chromadb` / `MagicCNKI` 的运行级别（required/optional）。
- `release-gate` 增加依赖预检报告（通过/降级/缺失）。
- 提供最小可复现环境脚本（本地/CI 一致）。

### C. 质量门禁增强
- Reviewer Gate 增加次级质量阈值（章节覆盖、最小字数、问题清单）。
- Export 前增加产物完整性校验（关键文件存在 + JSON schema 通过）。
- 回归报告输出统一结构（机器可读 JSON + 人类可读摘要）。

## 里程碑
- M1（1 周）：需求冻结 + ADR + 风险评审
- M2（1 周）：可靠性改造 + 单元/集成测试
- M3（1 周）：门禁增强 + 发布演练 + 验收

## 风险
1. 依赖安装条件差异导致 CI 与本地结论不一致。
2. 严格模式可能提高失败率，需要明确降级策略边界。
3. 门禁增强可能拉长发布周期，需要配套并行化测试策略。

## 验收标准
- `release-gate` 在标准环境下连续 3 次通过。
- 全量测试通过且无“静默降级”路径。
- 回滚演练完成并记录在 `ROLLBACK.md`。
