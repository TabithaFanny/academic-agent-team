# PaperGenius Pro — 产品全信息与用户交互流程

> **版本**: MVP 1.1  
> **目标**: 辅助产出 C 刊 (CSSCI) / 顶会 (ACL/NeurIPS/CHI) 级别学术论文  
> **定位**: 高质量辅助写作 + 强后处理（非纯端到端生成）

---

## 一、产品概览

### 1.1 一句话描述

**输入研究主题 → 自动文献调研 → 多 Agent 协作写作 → 输出可投稿级论文初稿 + 人工修改指南**

### 1.2 核心价值主张

| 痛点 | 解决方案 |
|------|----------|
| 找文献耗时 | CNKI + 国际数据库自动搜索，RAG 智能综述 |
| 写作卡壳 | 6 Agent 团队分工协作，章节循环迭代 |
| 引用不规范 | 自动 DOI/CNKI 验证，强制可溯源 |
| 格式繁琐 | 20+ 期刊 LaTeX 模板，一键编译 |
| AI 痕迹重 | AI 检测 + 降重 + 人工改写指南 |
| 学术诚信风险 | 全程可审计，强制人工参与关键环节 |

### 2.2 Agent 角色定义

| 角色 | 职责 | 核心工具 |
|------|------|----------|
| **Advisor** | 选题分析、研究空白识别、方向推荐 | topic_analyzer, gap_finder |
| **Researcher** | 文献搜索 (CNKI 优先)、RAG 综述生成 | search_cnki, search_arxiv, rag_query |
| **Data Analyst** | 数据分析、统计检验、图表生成 | run_statistics, generate_chart |
| **Writer** | 论文各章节撰写、引用注入 | write_section, inject_citations |
| **Reviewer** | 审稿评分、修改意见、质量把关 | review_section, score_quality |
| **Polisher** | 语言润色、降重改写、格式规范 | polish_text, reduce_similarity |

### 2.3 质量门禁 (Gates)

| Gate | 触发点 | 条件 | 失败处理 |
|------|--------|------|----------|
| **引用验证 Gate** | 文献调研后 | 验证率 ≥ 80% | 返回重新搜索 |
| **文献质量 Gate** | 文献调研后 | 总数 ≥ 30, CNKI ≥ 15 | 扩大搜索范围 |
| **AI 检测 Gate** | 每章节完成 | ai_score < 30% | 标记 + 人工改写 |
| **查重 Gate** | 每章节完成 | similarity < 20% | 降重建议 + Polisher 改写 |
| **内容质量 Gate** | 写作-审稿循环 | Reviewer 评分 ≥ 85 | 返回 Writer 重写 |

## 三、用户交互流程

### 3.0 两种运行模式

| 模式 | 适用场景 | 人工干预点 | 耗时 |
|------|----------|------------|------|
| **autopilot (默认)** | 稳定批量执行 | 最少人工介入 | 10-20 分钟 |
| **manual** | 人工审核优先 | 关键阶段人工确认 | 20-40 分钟 |

```bash
# 自动推进模式（默认）
$ paper-team start --topic "..." --journal "中文核心" --mode autopilot

# 人工介入模式
$ paper-team start --topic "..." --journal "中文核心" --mode manual
```

> 注：当前默认执行引擎已为 `pipeline_v2`；CLI 仍使用 `paper-team` 的 `autopilot/manual` 参数语义，并映射为 `express/standard`。如需回退可用 `--engine legacy`。

### 3.1 四个固定干预点 (Human-in-the-Loop)

| ID | 阶段 | 干预内容 | manual | autopilot |
|----|------|----------|--------|-----------|
| **H1** | 选题 | 从 3-5 个聚焦方向中选择 1 个 | ✅ 必须 | ✅ 必须 |
| **H2** | 文献 | 确认 Top 30 文献列表，可删除/补充 | ✅ 必须 | ⚠️ 默认自动继续 |
| **H3** | 写作 | Reviewer 打分后决定：继续迭代/手动修改 | ✅ 必须 | ⚠️ 默认自动继续 |
| **H4** | 导出 | 最终预览确认，查看修改指南 | ✅ 必须 | ✅ 必须 |

### 3.3 关键质量指标

| 指标 | 目标值 | 检测时机 | 未达标处理 |
|------|--------|----------|------------|
| **文献数量** | CNKI ≥ 15 + 国际 ≥ 10 | Phase 2 结束 | 自动扩大搜索 |
| **引用验证率** | ≥ 80% | Phase 2 结束 | 移除未验证引用 |
| **AI 检测得分** | < 30% | 每章节 + 最终 | 标记 + 改写建议 |
| **查重率** | < 20% | 每章节 + 最终 | 一键降重建议 |
| **Reviewer 评分** | ≥ 85/100 | Phase 3 每轮 | 继续迭代或人工修改 |

### 3.4 CNKI 降级策略

```
CNKI 搜索
    │
    ├── 成功 → 使用 CNKI 结果
    │
    └── 失败 (反爬虫/限流/网络错误)
            │
            ▼
        自动 Fallback
            │
            ├── Semantic Scholar (英文学术)
            └── OpenAlex (开放元数据)
```

## 五、学术诚信保障

### 5.1 不可妥协约束

| 约束 | 强制措施 |
|------|----------|
| 引用必须可溯源 | DOI/CNKI 链接强制验证 |
| AI 生成必须可审计 | 全程日志 + 版本追踪 |
| 查重率必须合规 | < 20% 强制检测 |
| 人工参与必须有 | 强制修改点标注 |

### 5.2 强制输出内容

**AI 辅助声明 (每篇论文必须包含)**:
```
本论文在撰写过程中使用了 AI 辅助工具进行：
- 文献检索与综述生成
- 初稿撰写与结构组织
- 语言润色与格式规范

所有核心观点、研究设计、数据分析及结论均由作者独立完成。
AI 生成内容比例约为 ___% (经检测)。
```

**人工修改指南 (每篇论文必须包含)**:
- 🔴 必须修改的高 AI 概率段落
- 🟡 建议润色的部分
- ✅ 可直接使用的部分
