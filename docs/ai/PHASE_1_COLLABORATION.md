## 4. 风险控制措施

### 4.1 范围蔓延防护

| 风险 | 控制措施 | 触发动作 |
|------|----------|----------|
| 功能膨胀 | MVP 范围锁定在 VISION.md | 新功能请求 → 升级人类 |
| 需求模糊 | PRD 每项标注 Clear/Ambiguous | Ambiguous → 升级人类 |
| 时间超限 | 每周进度检查 | 延期 > 3 天 → 升级人类 |

### 4.2 技术风险防护

| 风险 | 控制措施 | 触发动作 |
|------|----------|----------|
| 依赖冲突 | 新依赖需确认 | 编辑 pyproject.toml → C5 |
| 架构腐化 | 编码必须引用 ARCH.md 段落 | 绕过架构 → 拒绝执行 |
| 测试不足 | 新功能必须附带测试 | 无测试 → 阻塞合并 |

### 4.3 幻觉/质量防护

| 风险 | 控制措施 | 触发动作 |
|------|----------|----------|
| 代码幻觉 | 所有代码必须可执行 | 运行测试验证 |
| 文档幻觉 | 引用必须指向真实文件/段落 | 交叉检查 |
| API 幻觉 | 调用真实 API 验证 | Mock → Real 切换测试 |

### 4.4 回滚防护

| 场景 | 回滚策略 |
|------|----------|
| 编码错误 | Git revert 到上一个稳定 commit |
| 架构错误 | 回退到 Phase 3，重新设计 |
| 数据损坏 | 从备份恢复 sessions.db |
| 依赖冲突 | pip uninstall + 恢复 pyproject.toml |

## 5. 沟通协议

### 5.1 状态不一致处理

```
IF state.yaml 与 docs/ai/*.md 冲突:
    1. 立即停止所有操作
    2. 输出冲突详情
    3. 请求人类裁决
    4. 等待明确指令
```

### 5.2 阻塞问题升级

```
IF 遇到阻塞问题:
    1. 记录到 TASK_BOARD.md 阻塞区
    2. 标记为 🔴 Blocked
    3. 在输出契约中说明
    4. 提供建议选项
    5. 等待人类决策
```

### 5.3 错误恢复

```
IF AI 执行出错:
    1. 停止当前操作
    2. 记录错误到 SESSION_NOTES.md
    3. 尝试自动回滚（如果可安全执行）
    4. 报告错误详情
    5. 提供修复建议
```
