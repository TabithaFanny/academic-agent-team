### 1.2 关键路径缺失

#### ❌ 缺失 1：论文章节细粒度生成路径

**当前状态**：Writer Agent 一次性生成全文
**问题**：C 刊/顶会论文需要每个章节精雕细琢

**应有路径**：
```
Introduction 生成
    ↓ Reviewer 审查
    ↓ 通过
Related Work 生成 (基于 RAG 文献综述)
    ↓ Reviewer 审查引用密度
    ↓ 通过
Methodology 生成
    ↓ Reviewer 审查方法论严谨性
    ↓ 技术 Reviewer (如涉及算法)
    ↓ 通过
Experiments / Analysis 生成
    ↓ Data Analyst 生成图表
    ↓ Reviewer 审查实验设计
    ↓ 通过
Discussion 生成
Conclusion 生成
Abstract 生成 (基于全文)
```

#### ❌ 缺失 2：实验数据处理路径

**当前状态**：无数据分析 Agent
**问题**：社科论文需要问卷分析、统计检验；CS 论文需要 benchmark 对比

**应有路径**：
```
原始数据 (CSV/Excel/问卷)
    ↓
Data Analyst Agent
    ├── 描述性统计
    ├── 相关性/回归分析
    ├── t 检验 / ANOVA / χ² 检验
    └── 可视化图表 (matplotlib/seaborn)
    ↓
生成 Results 章节图表引用
```

#### ❌ 缺失 3：格式规范路径

**当前状态**：仅输出 Markdown
**问题**：C 刊/顶会都有严格格式要求

**应有路径**：
```
论文内容 (sections dict)
    ↓
目标期刊/会议识别
    ↓
LaTeX 模板选择
    ├── CSSCI 期刊格式
    ├── ACL/EMNLP 格式 (acl2025.sty)
    ├── NeurIPS 格式 (neurips_2025.sty)
    └── IEEE 格式
    ↓
LaTeX 编译
    ↓
PDF 输出 + Cover Letter
```

#### ❌ 缺失 4：引用规范路径

**当前状态**：RAG 返回 sources，但无引用格式化
**问题**：引用格式不规范是直接退稿理由

**应有路径**：
```
RAG 返回的引用
    ↓
DOI 验证 (CrossRef API)
    ↓
元数据补全 (作者、年份、期刊、卷期页)
    ↓
格式化 (GB/T 7714 / APA / IEEE)
    ↓
BibTeX 生成
    ↓
交叉引用检查 (正文引用 ↔ 参考文献列表)
```

### 3.3 差距总结

#### 🔴 致命差距（无法投稿）

1. **AI 生成检测规避**
   - 现状：完全没有
   - 必须：C 刊/顶会都在严打 AI 论文，纯 AI 生成 100% 被拒
   - 方案：
     - 集成 GPTZero/ZeroGPT API 自测
     - 输出"人工改写建议"而非直接输出终稿
     - 引导用户深度参与每个章节

2. **引用真实性验证**
   - 现状：仅 RAG 返回 sources
   - 必须：每条引用必须可追溯到真实文献
   - 方案：
     - CrossRef API 验证 DOI
     - 引用元数据强制补全
     - 无法验证的引用自动剔除

3. **查重降重**
   - 现状：完全没有
   - 必须：C 刊普遍要求 < 15-20%
   - 方案：
     - 集成 paperyy/维普 API（或自建语义相似度检测）
     - 高重复段落自动改写

4. **实验/数据支撑**
   - 现状：无数据分析能力
   - 必须：社科需要统计分析，CS 需要实验对比
   - 方案：
     - Data Analyst Agent
     - 代码沙箱执行统计分析
     - 图表自动生成

#### 🟡 严重差距（显著降低录用率）

5. **方法论薄弱**
   - 现状：Writer 自由发挥
   - 方案：按学科提供方法论模板（定性/定量/混合研究）

6. **格式不规范**
   - 现状：仅 Markdown
   - 方案：LaTeX 模板库 + 自动编译

7. **中文文献覆盖不足**
   - 现状：CNKI 在 Phase 2
   - 方案：CNKI MCP 提升优先级到 MVP

8. **创新点不清晰**
   - 现状：依赖 LLM 自由发挥
   - 方案："研究贡献提取"工具 + 明确的 Contribution 章节模板
